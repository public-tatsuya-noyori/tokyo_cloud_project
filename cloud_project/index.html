Publish/Subscribe/Clone on cloud (not using MQPs).

Evaluation

Publish/Subscribe/Clone on cloud (not using MQPs) has been being evaluated by using the following resources.

  • wis-jma to download all GTS data on Tokyo.
  • HP EliteDesk 800 G5 SFF as a local PC.
  • 3 wasabi cloud regions as follows.
    • ap-northeast-1 region of Wasabi cloud to publish open data and closed data.
    • us-east-1 region of Wasabi cloud to clone open data and closed data.
    • eu-central-1 region of Wasabi cloud to subscribe open data and closed data.

How to publish/subscribe/clone on cloud

  1. Install rclone

Reference

$ curl https://rclone.org/install.sh | sudo bash
  1. Configure rclone
$ vi $HOME/.config/rclone/rclone.conf
[wasabi]
type = s3
env_auth = false
access_key_id =
secret_access_key =
region =
endpoint = https://s3.ap-northeast-1.wasabisys.com
location_constraint =
acl = public-read
server_side_encryption =
storage_class =
[wasabi_us]
type = s3
env_auth = false
access_key_id =
secret_access_key =
region =
endpoint = https://s3.us-east-1.wasabisys.com
location_constraint =
acl = public-read
server_side_encryption =
storage_class =
[wasabi_eu]
type = s3
env_auth = false
access_key_id =
secret_access_key =
region =
endpoint = https://s3.eu-central-1.wasabisys.com
location_constraint =
acl = public-read
server_side_encryption =
storage_class =
:wq
  1. Install pub.sh/sub.sh/clone.sh

Download pub.sh/sub.sh/clone.sh

  1. Configure Cron

A sample configuration of Cron to subscribe p3 data files from ap-northeast-1 region is as follows.

crontab -e
* * * * * /path_to_sh/sub.sh --cron cache wasabi_open_data_p3 wasabi japan.meteorological.agency.open.data p3 16
:wq

A sample configuration of Cron to subscribe p1 data with urgent mode(execution interval = 15 seconds) from eu-central-1 region is as follows.

$ crontab -e
* * * * * /path_to_sh/sub.sh --cron --urgent cache wasabi_eu_open_data_p1 wasabi_eu japan.meteorological.agency.open.data.eu.central.1 p1 4
:wq

A sample configuration of Cron to subscribe Alphanumeric SYNOP/SHIP without Japan data from eu-central-1 region is as follows.

$ vi inclusive_pattern.txt
/alphanumeric/surface/synop/
/alphanumeric/surface/ship/
:wq
$ vi exclusive_pattern.txt
/R[JO][A-Z][A-Z]/
:wq
$ crontab -e
* * * * * /path_to_sh/sub.sh --cron cache wasabi_eu_open_p2_synop_ship_without_japan wasabi_eu japan.meteorological.agency.open.data.eu.central.1 p2 16 inclusive_pattern.txt exclusive_pattern.txt
:wq

Publish/Subscribe/Clone (not using MQPs)

publish to cloud

A reference implementation is pub.sh. The protocol to publish is as follows.

  1. upload data files.
  2. upload a file of a list of the uploaded data files to the directory of "4PubSub/p[1-9]". The name of the file is YYYYMMDDHHMMSS.txt.

subscribe from cloud

A reference implementation is sub.sh. The protocol to subscribe is as follows.

  1. download a current list of the index files in "4PubSub/p[1-9]"
  2. get the newly created list of the index files by comparing the previous list with the current list.
  3. download newly created index files from the directory of "4PubSub/p[1-9] ".
  4. download data files that are linked in each index files.

clone cloud to cloud

A reference implementation is clone.sh. The protocol to clone is as follows.

  1. download a current list of the index files in "4PubSub/p[1-9]"
  2. get the newly created list of the index files by comparing the previous list with the current list.
  3. download newly created index files from the directory of "4PubSub/p[1-9] ".
  4. copy data files that are linked in each index files from source cloud to destination cloud.
  5. upload a file of a list of the copied data files to the directory of "4PubSub/p[1-9]". The name of the file is YYYYMMDDHHMMSS.txt.

directory name convention

A reference implementation is met_pre_batch_to_cache.py. The convention is as follows.

  1. [bucket]/[indicator]/[format]/[category]/[subcategory]/[year+month+day+hour+minute]/
  2. [bucket] is bucket of cloud storage.
  3. [indicator] is uppercase.
  4. [format]/[category]/[subcategory] is lowercase.
  5. [format] is 'alphanumeric' or 'bufr' or 'crex' or 'grib' or 'iwxxm'.
